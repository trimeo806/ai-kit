# Codex (GitHub Copilot) Setup Instructions

To use this kit with Codex or GitHub Copilot, you first need to run the `.ps1` sync script to generate the appropriate format:

```powershell
# Run the sync script within the tri-ai-kit directory
.\scripts\sync-to-codex.ps1
```

Once synced, copy the Codex-compatible files into your project root:

```bash
# From the tri-ai-kit directory, copy into your target project
cp -r .github/agents /path/to/your-project/.github/
cp .github/copilot-instructions.md /path/to/your-project/.github/
cp WORKFLOW.md /path/to/your-project/
```

This ensures Copilot reads the necessary instructions and behaviors from `.github/copilot-instructions.md` and related agent files during its lifecycle.

**What gets installed:**
- `.github/agents/` — Codex-compatible agent descriptions
- `.github/copilot-instructions.md` — Global orchestration instructions
- `WORKFLOW.md` — 15-phase production delivery workflow reference

---

## Migrating or Syncing from Source (Claude)

This kit enforces a single source of truth from `.claude/`, which means your Codex skills and agents are automatically processed:

1. The current default target directory generated by the script is configured to `.github/agents/` and `.github/skills/`.
2. The `AGENTS.md` file itself is also automatically parsed and exported to `.github/copilot-instructions.md`.

The `.claude/` source remains unchanged. When the official Codex target directory format evolves, simply update the environment configuration variables at the top of the `scripts\sync-to-codex.ps1` file.
