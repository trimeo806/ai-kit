Review accessibility compliance for the detected platform and focus area.

## Aspect Files

| File | Coverage |
|------|----------|
| `references/ios-guidance-mode.md` | iOS guidance mode: real-time advice, code examples, response style rules |
| `references/android-guidance-mode.md` | Android guidance mode: Compose + Views/XML advice, code examples, TalkBack patterns |

**IMPORTANT:** Analyze the skills catalog and activate ONLY the skills needed for the detected platform.

## Platform Detection

See `a11y` skill for platform routing.

## Focus Detection

| Argument | Scope |
|----------|-------|
| `buttons` | Button labels, traits/roles, tap targets, states |
| `headings` | Heading traits/semantics, hierarchy, navigation order |
| `modals` | Focus trapping, modal attributes, dismiss actions |
| `forms` | Form labels, validation, error announcements, input types |
| `all` or _(empty)_ | **All areas** |

## Process

1. **Detect platform** — from argument or file context
2. **Get Git diff** — all changed files matching detected platform
3. **Scan focus area(s)** — using platform-specific skill rules
4. **Check known findings** — match against `.kit-data/a11y/known-findings.json` (if exists)
5. **Report** — unified findings with severity and fix suggestions
6. **Persist new findings** — same as audit-a11y step 8 (dedup by wcag+file_pattern+code_pattern, auto-assign ID, set `source: "review"`). Create `.kit-data/a11y/known-findings.json` if missing.
7. **Save report file** — write the full JSON output to `.kit-data/a11y/fixes/reviews/review-YYMMDD-HHMM.json` (create directories if missing). Print: `Report saved: .kit-data/a11y/fixes/reviews/review-YYMMDD-HHMM.json`

## Platform-Specific Checks

### iOS
- VoiceOver labels (`accessibilityLabel`), traits, grouping
- Dynamic type support, contrast ratios
- `accessibilityViewIsModal`, custom actions

### Android
- `contentDescription`, `semantics {}` modifiers
- Touch targets (48x48dp min), `stateDescription`
- `mergeDescendants`, `heading()`, focus order

### Web
- ARIA attributes, semantic HTML, landmark roles
- Keyboard navigation (`:focus-visible`, tab order)
- Form labels, `aria-live` regions, color contrast

## Rules

- Never skip findings — report all issues regardless of severity
- Platform-specific guidance takes priority over generic rules
- Always suggest concrete fixes, not just problem descriptions
